SECTION 2 — FEATURE ENGINEERING & PREPROCESSING
----------------------------------------------------------------

Q7. ColumnTransformer kyun use kiya?
A:  Ek hi step me alag-alag column types ko alag treatment
    dene ke liye:
    - Numeric → median impute + StandardScaler
    - Categorical → most_frequent impute + OneHotEncoder
    - Binary → passthrough (already 0/1)
    Yeh pipeline ko clean aur leakage-free rakhta hai.

Q8. bmi ki missing values ko kaise handle kiya?
A:  SimpleImputer(strategy='median') use ki — median
    outliers se robust hoti hai. Pipeline ke andar kiya
    taaki test data ka median train data se le (no leakage).

Q9. StandardScaler kyun zaroori hai?
A:  KNN aur SVM distance-based algorithms hain.
    Bina scaling ke age (0-82) aur avg_glucose_level (55-271)
    ka scale alag hoga jisse prediction biased ho jaayegi.
    Tree-based models (RF, DT) ko scaling ki zaroorat nahi.

Q10. OneHotEncoder me handle_unknown='ignore' kyun diya?
A:   Agar test set me koi unseen category aaye (e.g., gender
     = 'Other' sirf 1 row hai) toh error na aaye, uski jagah
     zero vector encode ho jaaye.

SECTION 5 — BASELINE MODELS
----------------------------------------------------------------

Q17. Baseline me 6 algorithms kyun use kiye?
A:   Har algorithm ek alag strategy use karta hai:
     - Logistic Regression: linear boundary
     - KNN: proximity-based voting
     - Naive Bayes: probabilistic, feature independence assume
     - Decision Tree: rule-based splits
     - Random Forest: ensemble of trees (bagging)
     - SVM: maximum margin hyperplane

Q18. Baseline results me accuracy 95% aur recall 0 kyun?
A:   Heavily imbalanced data (95% class-0) mein model simply
     sab kuch class-0 predict karta hai.
     → Accuracy = 95% (sab 0 bolo → 95% correct)
     → Recall for stroke = 0% (koi positive detect nahi)
     Yeh "accuracy paradox" hai — accuracy misleading metric
     hai imbalanced datasets me.

Q19. Evaluation metrics kaunse use kiye aur kya matlab?
A:   - Accuracy:  (TP+TN) / Total — imbalanced me misleading
     - Precision: TP / (TP+FP) — positive predictions me se kitne sach?
     - Recall:    TP / (TP+FN) — actual positives me se kitne pakde?
     - F1 Score:  2 * (P*R)/(P+R) — precision aur recall ka balance
     - ROC-AUC:   discrimination ability (1.0 = perfect, 0.5 = random)

     Healthcare me Recall sabse critical hai — miss karna
     (FN) zyada dangerous hai galat alarm (FP) se.

Q20. NaiveBayes ka recall dusron se zyada kyun tha?
A:   GaussianNB probabilistic hai aur minority class ke
     prior probability ko naturally consider karta hai.
     Yeh soft predictions deta hai — harder decision boundary
     nahi banata — isliye stroke cases pakad leta hai.

----------------------------------------------------------------
SECTION 6 — CLASS IMBALANCE
----------------------------------------------------------------

Q21. class_weight='balanced' kya karta hai?
A:   Minority class (stroke=1) ko automatically zyada weight
     deta hai formula se:
     weight = total_samples / (n_classes * class_count)

     Stroke=0 weight ~ 0.52, Stroke=1 weight ~ 10.26
     Model ab minority class ki errors ko zyada penalize
     karta hai → recall improve hota hai.

Q22. class_weight='balanced' ke baad kya change hua?
A:   Before: Recall ~0, Accuracy ~95%
     After:  Recall ~0.74 (LogReg_bal, SVM_bal), Accuracy ~76%
     Tradeoff: Recall badha, Accuracy thodi giri —
     yeh healthcare me acceptable hai.

Q23. SMOTE kya hai aur kab use karein?
A:   Synthetic Minority Over-sampling Technique.
     Minority class ke synthetic samples banata hai
     nearest neighbors interpolate karke.
     Use karein: KNN/NaiveBayes jaise models ke liye
     jo class_weight support nahi karte.
     Library: imbalanced-learn (pip install imbalanced-learn)

Q24. class_weight vs SMOTE — kaunsa better?
A:   - class_weight: fast, no data augmentation, pipeline me easy
     - SMOTE: actual data create karta hai, overfitting ka risk
     Dataset pe depend karta hai. Stroke jaise small dataset
     pe class_weight prefer karo, SMOTE bhi valid option hai.

SECTION 7 — GRIDSEARCHCV
----------------------------------------------------------------

Q25. GridSearchCV kya karta hai?
A:   Hyperparameter combinations ko systematically try karta hai
     cross-validation ke saath. Best combination woh jo
     validation metric (scoring) maximize kare.

Q26. Scoring='f1' kyun rakha, 'accuracy' kyun nahi?
A:   Imbalanced data me accuracy trivially high hoti hai
     (sab 0 bol do = 95%). F1 precision aur recall dono
     consider karta hai — minority class performance reflect
     karta hai accurately.

Q27. Best params kya mile?
A:   clf__max_depth=10, clf__min_samples_split=5,
     clf__n_estimators=200
     Best CV F1 = 0.281

Q28. cv=5 ka matlab kya hai?
A:   5-Fold Cross Validation — training data ko 5 parts me
     baanto, 4 pe train karo, 1 pe validate karo.
     5 baar rotate karo. Average score = CV score.
     Overfitting detect karne ka reliable tarika.

Q29. n_jobs=-1 kya karta hai?
A:   Saare available CPU cores use karta hai parallel me.
     GridSearch bahut slow hoti hai ek core pe —
     n_jobs=-1 se training time kaafi kam ho jaata hai.

----------------------------------------------------------------
SECTION 8 — ENSEMBLE METHODS
----------------------------------------------------------------

Q30. Bagging kya hai? Uska example?
A:   Bootstrap Aggregating — multiple models independently
     train karo (random data subsets pe), predictions combine karo.
     Example: RandomForest (decision trees ka bagging)
     Advantage: variance reduce karta hai, overfitting kam.

Q31. Boosting kya hai? Bagging se kaise alag?
A:   Sequential learning — pehla model train hota hai,
     jo galat predictions karta hai unpe zyada focus karo,
     agli model usi pe focus karta hai.
     Examples: AdaBoost, GradientBoosting, XGBoost
     Difference: Bagging = parallel, Boosting = sequential
     Boosting bias reduce karta hai, Bagging variance.

Q32. AdaBoost vs GradientBoosting?
A:   AdaBoost: misclassified samples ka weight badhata hai
     GradientBoosting: residual errors minimize karta hai
                       (gradient descent in function space)
     GradientBoosting generally zyada powerful hai.

Q33. Stacking kya hai?
A:   Multiple diverse models (KNN, SVM, DT) ke predictions
     ko input ki tarah use karo, ek meta-model (LogisticReg)
     train karo final prediction ke liye.
     Diverse base models ko combine karta hai smartly.

Q34. Ensemble results me kya interesting tha?
A:   - Stacking: highest Recall (0.70) — diverse models combine
     - Bagging: best F1 among ensembles (0.280)
     - GradientBoosting: high accuracy (0.951) but recall ~0
       (imbalance handle nahi kiya)
     Ensemble alone kaafi nahi — imbalance bhi handle karo.

SECTION 10 — GENERAL CONCEPTS
----------------------------------------------------------------

Q39. Overfitting kya hai? Kaise detect karein?
A:   Model training data pe bahut accha kare lekin test pe
     kharab — generic patterns nahi seekhe, noise seekha.
     Detect: Train accuracy >> Test accuracy
     Fix: Cross-validation, regularization, pruning, more data.

Q40. Underfitting kya hai?
A:   Model train aur test dono pe kharab kare — too simple.
     Fix: Complex model use karo, more features, less regularization.

Q41. Precision aur Recall me tradeoff kab important hota hai?
A:   - Cancer/Stroke detection: Recall maximize karo
       (FN = miss karna = dangerous)
     - Spam filter: Precision maximize karo
       (FP = important email spam me = irritating)
     - F1 = equal balance chahiye toh

Q42. Pipeline ka fayda kya hai?
A:   1. Data leakage prevent — test data se fit nahi hoti
     2. Code clean — ek hi object me sab steps
     3. GridSearchCV compatible
     4. Production deployment easy — ek hi object save karo

Q43. Feature importance kaise dekh sakte hain RF me?

     importances = rf_model.feature_importances_
     Common top features: age, avg_glucose_level, bmi
